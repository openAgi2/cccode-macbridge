## 3. Known Issues (遗留问题)

1. **（当前阻塞）持久化 transcript-index 缓存让修复版代码不生效**：旧 bug runtime 生成的 codex index（25-span）被修复版 runtime 复用 → replay 仍 fallback → close 1009。**删除缓存重建即可**（见 Next Steps）。
2. **（设计缺陷，长期修）extractor 逻辑变更不触发持久化 index 失效**：fingerprint 只含 mtime+size + IndexVersion，span 提取算法改了不会让旧 index 作废。应在 IndexVersion 之外加「extractor/schema 指纹」字段，extractor 改动时 bump，使旧 index 被丢弃重建。这次边界修复就栽在这上面。
3. **Relay 帧限制 < 7MB**（7MB 触发 close 1009）：byte budget 已调 1MB 规避；但 1MB 让大会话首页只显示几条消息（需 Stage 2 向后翻页才能看更早历史）。
4. **iPhone 走公网 Relay 非 LAN 直连**（`transport_route=relay`）：列表 ~10s 的根因（后端实际 23ms）。属配对/网络选择，非分页 bug。
5. **cccode-ios 无 Phase 0 iOS 埋点**（SessionLoadingTracer）：iOS 端精细测量回归无法做，需在新仓库补建（或在 exec-plan 把 phase0-ios 降级/重做）。
6. **`unknown device dev_1fa33750`** 警告：16 Pro 设备凭证在 Bridge 受信列表半残（后认证通过）；若连接不稳需重新配对。
